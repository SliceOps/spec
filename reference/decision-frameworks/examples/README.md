# Technical Decision Analysis Examples

These examples show how to use the technical decision analysis template without turning it into a mechanical checklist.

Use them as reference shapes:

- [`api-endpoint-example.md`](api-endpoint-example.md) - focused analysis for an API endpoint.
- [`react-feature-example.md`](react-feature-example.md) - focused analysis for a frontend feature.
- [`data-migration-example.md`](data-migration-example.md) - full analysis for a data migration.

The pattern is always:

1. Load accepted decisions.
2. Classify the slice relationship.
3. Mark dimension materiality.
4. Load only the stack packs that match material dimensions.
5. Select the adequate path.
6. Attach evidence and revisit triggers.
